# üîµ **1. Set a static IP, gateway, DNS using nmcli**

**Example Scenario (modify as needed):**

* Interface: **ens33**
* Static IP: **192.168.1.50/24**
* Gateway: **192.168.1.1**
* DNS: **8.8.8.8**

---

### **Steps**

```bash
sudo nmcli con mod ens33 ipv4.addresses 192.168.1.50/24
sudo nmcli con mod ens33 ipv4.gateway 192.168.1.1
sudo nmcli con mod ens33 ipv4.dns 8.8.8.8
sudo nmcli con mod ens33 ipv4.method manual
```

Apply changes:

```bash
sudo nmcli con down ens33
sudo nmcli con up ens33
```

---

### **Verify**

```bash
ip a show ens33
ip r
nmcli dev show ens33 | grep DNS
ping -c 3 8.8.8.8
```

---

# üîµ **3. Create an additional network connection profile**

**Example: create a second profile named `office-net` for interface ens33.**

### **Steps**

```bash
sudo nmcli con add con-name office-net ifname ens33 type ethernet ipv4.method manual ipv4.addresses 192.168.10.50/24 ipv4.gateway 192.168.10.1 ipv4.dns 1.1.1.1
```

Do NOT activate it automatically unless required.

---

### **Verify**

```bash
nmcli con show
```

You should see:

* `ens33`
* `office-net`

---

# üîµ **4. Enable and verify network autoconnect behavior**

You must be able to **enable** or **disable** autoconnect on any connection profile.

---

## **Enable autoconnect**

```bash
sudo nmcli con mod ens33 connection.autoconnect yes
```

## **Disable autoconnect**

```bash
sudo nmcli con mod office-net connection.autoconnect no
```

---

### **Verify**

```bash
nmcli con show ens33 | grep autoconnect
nmcli con show office-net | grep autoconnect
```

Expect:

* `ens33` ‚Üí yes
* `office-net` ‚Üí no

# üî• **1Ô∏è‚É£ Combined Network Configuration Scenario (RHCSA-Style)**

You are the administrator of a RHEL server with one interface **ens33**.
The system should operate inside a corporate environment with three network profiles.

Complete all tasks exactly as specified.

---

## **üî∑ Tasks ‚Äî Configuration**

### **A. Configure the primary network profile**

* Profile name: **corp-net**
* Interface: **ens33**
* Static IP: **10.50.20.25/24**
* Gateway: **10.50.20.1**
* DNS: **10.50.20.53**
* Autoconnect: **enabled**

### **B. Configure a secondary network profile**

* Profile name: **maintenance-net**
* Interface: **ens33**
* Static IP: **172.30.0.25/24**
* Gateway: **172.30.0.1**
* DNS: **1.1.1.1**
* Autoconnect: **disabled**

### **C. Configure a backup profile**

* Profile name: **backup-net**
* Interface: **ens33**
* DHCP enabled
* Autoconnect: **enabled**, but priority lower than corp-net

(Meaning: `connection.autoconnect-priority 10`)

### **D. Change system hostname**

* Hostname: **node3.corp.internal**

---

## **üî∑ Expected Solution Commands (No Wasted Steps)**

### **A. corp-net**

```bash
nmcli con add con-name corp-net ifname ens33 type ethernet ipv4.method manual \
ipv4.addresses 10.50.20.25/24 ipv4.gateway 10.50.20.1 ipv4.dns 10.50.20.53
nmcli con mod corp-net connection.autoconnect yes
```

### **B. maintenance-net**

```bash
nmcli con add con-name maintenance-net ifname ens33 type ethernet ipv4.method manual \
ipv4.addresses 172.30.0.25/24 ipv4.gateway 172.30.0.1 ipv4.dns 1.1.1.1
nmcli con mod maintenance-net connection.autoconnect no
```

### **C. backup-net**

```bash
nmcli con add con-name backup-net ifname ens33 type ethernet ipv4.method auto
nmcli con mod backup-net connection.autoconnect yes
nmcli con mod backup-net connection.autoconnect-priority 10
```

### **D. Hostname**

```bash
hostnamectl set-hostname node3.corp.internal
```

---

nmcli con show corp-net | grep -E 'ipv4|autoconnect'
nmcli con show maintenance-net | grep autoconnect

# üî• **2Ô∏è‚É£ RHCSA-Style Network Troubleshooting Exam**

You are asked to fix all network-related issues on a malfunctioning system.

Below is the exam scenario.

---

## **üî∑ Scenario ‚Äî What‚Äôs Broken**

1. The system cannot reach external networks.
2. DNS resolution fails completely.
3. On boot, the wrong network profile activates.
4. A second profile called `test-net` exists but is misconfigured.
5. A static route to **10.80.0.0/16** is missing.
6. The hostname does not match corporate policy.

Your job: **Fix every issue.**

---

## **üî∑ Known Facts**

* Interface: **ens33**
* Correct corporate IP: **192.168.5.100/24**
* Correct gateway: **192.168.5.1**
* DNS servers: **8.8.8.8, 8.8.4.4**
* Correct hostname: **infra-node2.corp.local**
* Required static route: network **10.80.0.0/16**, via gateway **192.168.5.254**
* The active profile should be **corp-fixed**, not `test-net`.

---

## **üî∑ Troubleshooting Tasks**

### **1. Fix IP configuration**

Check current config:

```bash
nmcli dev show ens33
```

If incorrect:

```bash
nmcli con mod corp-fixed ipv4.addresses 192.168.5.100/24
nmcli con mod corp-fixed ipv4.gateway 192.168.5.1
nmcli con mod corp-fixed ipv4.method manual
```

### **2. Fix DNS**

```bash
nmcli con mod corp-fixed ipv4.dns "8.8.8.8 8.8.4.4"
```

### **3. Correct autoconnect behavior**

Disable test profile:

```bash
nmcli con mod test-net connection.autoconnect no
```

Enable correct one:

```bash
nmcli con mod corp-fixed connection.autoconnect yes
```

### **4. Fix the static route**

```bash
nmcli con mod corp-fixed +ipv4.routes "10.80.0.0/16 192.168.5.254"
```

### **5. Hostname correction**

```bash
hostnamectl set-hostname infra-node2.corp.local
```

### **6. Bring the fixed profile up**

```bash
nmcli con down test-net
nmcli con up corp-fixed
```

---

## **üî∑ Troubleshooting Verification**

```bash
ip a
ip r
nmcli con show corp-fixed
ping -c 3 8.8.8.8
cat /etc/resolv.conf
hostnamectl
```
